🚀 Introduction to Java

BEGINNER · FOUNDATION

Java is a general-purpose, object-oriented, platform-independent language: source code compiles to bytecode that runs on the Java Virtual Machine (JVM), so the same program can run on Windows, macOS, or Linux without changes.

▶ Bro Code — Java Full CourseGeeksforGeeks: Introduction to Java
KEY CONCEPTS
  • Java code compiles to bytecode, then the JVM runs it — "write once, run anywhere"
  • Java is strongly typed, case-sensitive, and fully object-oriented
  • Every application needs a public static void main(String[] args) entry point
  • The JDK (Java Development Kit) is what you install to write and run Java

🧮 Variables

BEGINNER · FOUNDATION

A variable is a named container that stores a value of a specific type. Java is statically typed, so you declare the type up front — once created, that variable can only ever hold values of that type.

▶ Bro Code — Java Full CourseGeeksforGeeks: Java Variables

Swapping two variables' values is a classic beginner exercise. The safest approach uses a third, temporary variable to hold one value while the swap happens, so no data gets overwritten before it's copied.

▶ Bro Code — Java Full CourseGeeksforGeeks: Swap Two Numbers

  • A simple exercise for building comfort with variable assignment

📦 Objects & Classes (OOP)

BEGINNER · FOUNDATION

A class is a blueprint or template — it defines what properties (fields) something will have and what it can do (methods). An object is an actual instance built from that blueprint using the new keyword. You can create as many independent objects as you want from a single class, and each one holds its own data.

▶ Bro Code — Java Full CourseGeeksforGeeks: Classes & Objects
KEY CONCEPTS
  • A class is a blueprint that defines properties (fields) and behavior (methods)
  • An object is an instance of a class, created with the new keyword
  • Fields hold state; methods define behavior
  • Multiple objects can be created from a single class

BEGINNER · INITIALIZATION

A constructor is a special method whose name exactly matches the class name and has no return type. Whenever an object is created with new, the constructor runs automatically and sets up that object's initial state.

🎯 Variable Scope

BEGINNER · FOUNDATION

Scope determines where a variable can be seen and used. A local variable only exists inside the method where it's declared. An instance variable is declared in the class, and each object gets its own separate copy. A static (class) variable is shared by every object of that class.

▶ Bro Code — Java Full CourseGeeksforGeeks: Variable Scope
KEY CONCEPTS
  • Local variables — declared inside a method, only available there
  • Instance variables — declared in a class, one separate copy per object
  • Class/static variables — shared across every instance
  • Use this to tell a local variable apart from an instance variable with the same name

⚡ Static Keyword

INTERMEDIATE · SHARED STATE

A static field or method belongs to the class itself, not to any one object. That means every object shares the exact same copy of a static field, and a static method can be called directly using the class name, without creating an object first.

▶ Bro Code — Java Full CourseGeeksforGeeks: static Keyword
KEY CONCEPTS
  • A static member belongs to the class, not to a specific instance
  • All objects share the same value of a static field
  • Static methods can be called directly as ClassName.methodName()
  • A static method can't directly access non-static (instance) members

Inheritance, Encapsulation, Abstraction, and Polymorphism — the four pillars that make code reusable and maintainable.

👨‍👩‍👧 Inheritance

PILLAR · REUSABILITY

Using the extends keyword, one class (a child/subclass) inherits the fields and methods of another class (its parent/superclass). Instead of rewriting the same code, you keep it in one common parent class and let every child class reuse it.

▶ Bro Code — Java Full CourseGeeksforGeeks: Inheritance
KEY CONCEPTS
  • extends gives a child class the parent's fields and methods
  • Reduces duplicate code and increases reusability
  • Represents an "is-a" relationship — e.g. Dog is-a Animal
  • Java supports single inheritance (one parent class only)
24 / 32

🔁 Method Overriding

PILLAR · CUSTOMIZATION

A child class can redefine a method it inherited from its parent, keeping the same name and parameters — this is called overriding. It lets a subclass replace the parent's default behavior with its own, more specific version.

▶ Bro Code — Java Full CourseGeeksforGeeks: Method Overriding
KEY CONCEPTS
  • An overriding method must match the parent's name and parameters exactly
  • Using @Override is good practice and catches mistakes early
  • At runtime, the version matching the object's actual type is called
  • Different from overloading — overloading happens in one class, overriding across parent/child

🔼 super Keyword

PILLAR · PARENT ACCESS

The super keyword lets a child class reach directly into its parent's constructor, methods, or fields. It's especially useful inside an overridden method when you still want to run the parent's original version via super.methodName().

▶ Bro Code — Java Full CourseGeeksforGeeks: super Keyword
KEY CONCEPTS
  • super() calls the parent class's constructor
  • super.method() calls the parent's original version of a method
  • If a child doesn't call it explicitly, Java calls super() for you
  • Lets you keep parent behavior even after overriding it

🧩 Abstraction

PILLAR · SIMPLICITY

Abstraction means showing only what's necessary and hiding the complicated implementation underneath. In Java, this is built using abstract class and abstract method — the parent class declares "what" needs to happen, and each child class decides "how" it actually happens.

▶ Bro Code — Java Full CourseGeeksforGeeks: Abstraction
KEY CONCEPTS
  • You can't create an object directly from an abstract class
  • An abstract method has no body — subclasses must implement it
  • The goal is hiding complexity and showing only the essentials
  • Guarantees a common structure/contract across subclasses
27 / 32

🔐 Access Modifiers

PILLAR · VISIBILITY CONTROL

Access modifiers decide where a class, field, or method can be reached from. Java has four levels — public (anywhere), protected (same package + subclasses), default / package-private (same package only), and private (only inside that class).

▶ Bro Code — Java Full CourseGeeksforGeeks: Access Modifiers
KEY CONCEPTS
  • public — accessible from anywhere
  • private — accessible only inside its own class
  • protected — accessible from the same package and subclasses
  • default (no modifier) — accessible only within the same package

🛡️ Encapsulation

PILLAR · DATA PROTECTION

Encapsulation means keeping a class's fields private, blocking direct outside access, and instead exposing controlled access through getter/setter methods — so invalid or unsafe values never sneak in.

▶ Bro Code — Java Full CourseGeeksforGeeks: Encapsulation
KEY CONCEPTS
  • Keeping fields private blocks direct outside access
  • getX() / setX() methods provide controlled access
  • Validation inside a setter can stop invalid data from being set
  • Improves data protection and long-term maintainability

📋 Copy Objects

ADVANCED · MEMORY MODEL

Writing obj2 = obj1 doesn't create a new object — it just copies the reference to the same one. To get a truly independent copy, you need to build a new object and copy the values over yourself (a shallow copy, or a deep copy for nested objects).

▶ Bro Code — Java Full CourseGeeksforGeeks: Object Cloning
KEY CONCEPTS
  • = copies a reference, not a new object
  • Changing one variable's object affects the other, since they point to the same thing
  • Getting an independent copy means building a new object and copying its fields
  • Know the difference between a shallow copy and a deep copy when objects are nested
30 / 32

🔌 Interfaces

ADVANCED · CONTRACTS

An interface is a fully abstract "contract" — it only lists method signatures, with no implementation at all. Any class that implements it must provide those methods. This gives Java a way to get multiple-inheritance-like flexibility.

▶ Bro Code — Java Full CourseGeeksforGeeks: Interfaces
KEY CONCEPTS
  • Interface methods are abstract by default (pre-Java 8 style)
  • A class can implements more than one interface at once
  • Guarantees that certain methods will exist on the implementing class
  • Unlike an abstract class, an interface is fully abstract and supports multiple implementation
31 / 32

🎭 Polymorphism

ADVANCED · MANY FORMS

Polymorphism literally means "many forms." It means the same method call, or the same interface, can behave differently depending on the object behind it. An Animal reference could actually hold a Dog or a Cat, and calling sound() on it behaves differently each time.

▶ Bro Code — Java Full CourseGeeksforGeeks: Polymorphism
KEY CONCEPTS
  • Same method/operation behaves differently across different objects
  • Compile-time polymorphism — method overloading
  • Runtime polymorphism — method overriding
  • A parent-type reference can hold and handle a child object
32 / 32

✨ Dynamic Polymorphism

ADVANCED · RUNTIME BINDING

Also called runtime polymorphism, this is what method overriding actually produces. Which method runs isn't decided when the program compiles — it's decided while the program is running, based on the object's real class, not the reference type pointing to it.

▶ Bro Code — Java Full CourseGeeksforGeeks: Runtime Polymorphism
KEY CONCEPTS
  • Which method runs is resolved at runtime, not at compile time
  • The correct overridden method is chosen based on the object's actual type
  • Also known as "dynamic method dispatch"
  • Core to designing flexible, extensible real-world systems
